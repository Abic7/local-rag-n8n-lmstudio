QUICK START (15 Minutes)
────────────────────────────────────────────────────────────────────────────

STEP 1: Install Ollama
  Windows: Download from https://ollama.ai or run: winget install Ollama.Ollama
  macOS:   brew install ollama
  Linux:   curl https://ollama.ai/install.sh | sh

STEP 2: Download Models
  ollama pull gemma:7b
  ollama pull qwen:text-embedding-qwen3-embedding-8b
  (Wait 15-20 minutes for downloads)

STEP 3: Start Ollama Service
  Windows: Starts automatically (check system tray)
  macOS/Linux: ollama serve

STEP 4: Start n8n
  docker run -p 5678:5678 n8n/n8n

STEP 5: Import Workflow
  http://localhost:5678 → File → Import → rag-workflow.json

STEP 6: Test
  curl -X POST http://localhost:5678/webhook/rag \
    -H "Content-Type: application/json" \
    -d '{"question": "What is RAG?"}'

TROUBLESHOOTING
────────────────────────────────────────────────────────────────────────────

"Ollama not running"
  → ollama serve (macOS/Linux)
  → Check Windows system tray or restart Ollama service

"Model not found: gemma:7b"
  → ollama pull gemma:7b

"Out of memory"
  → Use smaller model: ollama pull gemma:2b
  → Close other applications
  → Enable GPU (if available)

"Very slow (20+ seconds)"
  → Likely running on CPU. Check GPU status: nvidia-smi
  → Install NVIDIA CUDA 11.8+ or Apple Metal (auto on macOS)

"Empty answer"
  → Verify Ollama running: curl http://localhost:11434/api/tags
  → Check model names in n8n Code nodes (should be "gemma:7b")

DEPLOYMENT OPTIONS
────────────────────────────────────────────────────────────────────────────

LOCAL (This PC/Mac):
  - Pros: Simplest setup, full control, no infrastructure
  - Cons: Limited scalability, tied to single machine
  - Best for: Demos, internal tools, development

DOCKER CONTAINER:
  - Pros: Reproducible, portable, version control
  - Cons: Requires Docker, slightly more complex
  - Best for: Staging/production, team deployment

DOCKER COMPOSE (Ollama + n8n + Qdrant):
  - Pros: Full stack, GPU support, persistent volumes
  - Cons: Most complex setup
  - Best for: Production, multi-component systems
